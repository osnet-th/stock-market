# ECOS 경제지표 차트 시각화

**날짜:** 2026-04-02
**상태:** 브레인스톰 완료

## 무엇을 만드는가

현재 테이블로만 보여주는 ECOS 경제지표 데이터를 **라디오 버튼 토글**로 표/그래프 뷰를 전환할 수 있게 하고, 그래프 뷰에서는 카테고리 내 각 지표의 **cycle별 추이를 개별 라인 차트**로 보여준다.

## 왜 이 접근인가

- `ecos_indicator` 테이블에 이미 cycle별 히스토리 데이터가 축적되고 있으나, 현재는 최신값+이전값만 테이블에 표시
- 지표의 시간에 따른 변화 추이를 시각적으로 파악하고 싶은 니즈
- Chart.js가 이미 프로젝트에 포함되어 있어 추가 라이브러리 불필요

## 핵심 결정

| 결정 사항 | 선택 | 이유 |
|-----------|------|------|
| 뷰 전환 방식 | 라디오 버튼 (표/그래프) | 직관적 UI, 기존 테이블 유지 |
| 차트 구성 | 카테고리 내 지표별 개별 라인 차트 | 단위 충돌 없이 깔끔한 비교 |
| 차트 타입 | Chart.js 라인 차트 | 추이 파악에 적합, 이미 프로젝트에 포함 |
| 데이터 조회 방식 | 카테고리 일괄 히스토리 API (방식 B) | API 호출 1회로 효율적 |
| 히스토리 범위 | 전체 히스토리 | 저장된 모든 cycle 데이터 표시 |

## 구현 범위

### 백엔드
- `GET /api/economics/indicators/history?category={CATEGORY}` 엔드포인트 추가
- `ecos_indicator` 테이블에서 카테고리에 해당하는 지표들의 전체 히스토리 조회
- 응답: 지표별로 그룹핑된 cycle-value 배열

### 프론트엔드
- ECOS 섹션에 라디오 버튼 (표 보기 / 그래프 보기) 추가
- 그래프 뷰: 카테고리 내 각 지표마다 소형 라인 차트를 카드 형태로 나열
- Chart.js Line Chart 사용 (X축: cycle, Y축: dataValue)
- 카테고리 전환 시 히스토리 데이터 로드

## 주의사항
- cycle 형식이 지표마다 다를 수 있음 ("2024.03", "2024Q1" 등) — X축 라벨 처리 필요
- 지표 수가 많은 카테고리에서 차트가 너무 많아질 수 있음 — 스크롤로 처리
