# 관심 지표 대시보드 브레인스토밍

**날짜:** 2026-04-15
**상태:** 브레인스토밍 완료

## 무엇을 만드는가

메인 대시보드에 **관심 지표 시스템**을 도입한다. 사용자가 국내(ECOS)/글로벌 경제지표 페이지에서 개별 지표를 "관심 지표"로 등록하면, 메인 대시보드에서 해당 지표를 카드 형태로 바로 확인할 수 있다.

### 대시보드 구성

```
┌─────────────────────────────────────────┐
│  🇰🇷 국내 경제지표                        │
├─────────────────────────────────────────┤
│  최신 업데이트                            │
│  기준금리 3.5% (04-15) | CD금리 3.8%...  │
├─────────────────────────────────────────┤
│  ⭐ 관심 지표                             │
│  ◀ [spread-card][spread-card][...] ▶   │
└─────────────────────────────────────────┘

┌─────────────────────────────────────────┐
│  🌍 글로벌 경제지표                       │
├─────────────────────────────────────────┤
│  최신 업데이트                            │
│  GDP 한국 2.1% (04-14) | ...            │
├─────────────────────────────────────────┤
│  ⭐ 관심 지표                             │
│  ◀ [spread-card][spread-card][...] ▶   │
└─────────────────────────────────────────┘
```

## 이 접근 방식을 선택한 이유

- **상하 분리(국내/글로벌)**: 두 지표 체계가 데이터 구조와 출처가 다르므로 명확히 구분
- **상단 요약 + 하단 관심지표**: 최신 업데이트로 변화를 빠르게 파악하고, 관심 지표로 상세 확인
- **서버 DB 저장**: JWT 기반 사용자별 관리, 기기 간 동기화 가능
- **기존 spread-card 재활용**: 일관된 UI, 추가 디자인 불필요

## 핵심 결정 사항

| 결정 | 선택 | 이유 |
|------|------|------|
| 용어 | "관심 지표" | 투자/경제 맥락에 자연스러움 |
| 저장 위치 | 서버 DB (MariaDB) | 사용자별 관리, 기기 간 동기화 |
| 등록 단위 | 개별 지표 | 국내: 기준금리, CD금리 등 / 글로벌: GDP, 인플레이션 등 지표 타입 단위 |
| 최신 요약 | 히스토리 수집 시 업데이트된 지표 | 지표명 + 날짜 + 값 표시 |
| 카드 형태 | 기존 spread-card 동일 | 게이지바 + 상태뱃지 포함 |
| 레이아웃 | 상단 요약 + 하단 관심지표 슬라이드 | 국내/글로벌 각각 영역 분리 |
| 슬라이드 | CSS overflow-x + 좌우 버튼 | 관심지표가 많아질 때 대응 |
| 클릭 이동 | 없음 | 대시보드에서 바로 확인, 별도 네비게이션 불필요 |

## 구현 범위

### 백엔드
- `favorite_indicator` 테이블 신규 생성 (사용자ID, 지표타입 구분, 지표코드)
- 관심 지표 CRUD API (등록/해제/조회)
- 최신 업데이트 지표 조회 API (히스토리 수집 시 변경된 지표)

### 프론트엔드
- ECOS/글로벌 페이지: 각 지표 옆 관심 등록 ★ 토글 버튼
- 메인 대시보드: 국내/글로벌 영역 분리 + 요약 섹션 + 관심 지표 슬라이드
- 좌우 스크롤 구현 (CSS `overflow-x: auto` + 화살표 버튼)
