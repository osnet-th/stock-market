# Brainstorm: 포트폴리오 재무제표 UI 개선

**Date:** 2026-03-19
**Status:** Draft

---

## What We're Building

포트폴리오 화면에서 재무제표 정보를 더 직관적으로 파악할 수 있도록 UI를 개선한다.

### 현재 문제점

1. **금액 가독성 부족**: 전체 금액이 단위 없이 표시되어 자릿수를 직접 세어야 함 (예: `1,234,567,890`)
2. **추세 파악 어려움**: 당기/전기/전전기 데이터가 테이블 컬럼으로만 비교되어 증감 추세가 한눈에 안 들어옴
3. **공간 제약**: 카드 내부 펼침식 패널에 재무 정보가 담겨있어 가로/세로 공간이 부족하고, 재무 계정이 길어 한 화면에 다 보이지 않음

### 개선 방향

- **우측 슬라이드 패널**: 카드에서 버튼 클릭 시 화면 우측에서 슬라이드인하는 넓은 패널로 재무 정보 표시
- **억원 단위 포맷팅**: `1,234,567,890원` → `12.3억` 형태로 금액 표시
- **바 차트**: 주요 계정의 당기/전기/전전기 금액을 세로 바 차트로 시각적 비교
- **8가지 메뉴 유지**: 현재 제공 중인 전체 메뉴를 탭으로 유지

---

## Why This Approach

### 우측 슬라이드 패널을 선택한 이유

- 페이지 이동 없이 빠르게 재무 정보 확인 가능
- 포트폴리오 목록이 배경에 살짝 보여 현재 컨텍스트 유지
- 패널 폭을 60~70%로 잡으면 차트 + 테이블을 충분히 배치 가능
- Alpine.js의 `x-show` + CSS transition으로 간단하게 구현 가능

### 억원 단위를 선택한 이유

- 한국 재무제표에서 가장 익숙한 단위
- 자릿수를 세지 않아도 금액 규모를 즉시 파악 가능
- 소수점 1자리(12.3억)로 정밀도와 가독성의 균형

### 바 차트를 선택한 이유

- 재무제표에서 연도별 비교에 가장 직관적인 차트 형태
- Chart.js가 이미 프로젝트에 포함되어 있어 추가 의존성 불필요
- 계정별로 당기/전기/전전기 3개 바를 그룹으로 묶어 비교

## Key Decisions

| 결정 사항 | 선택 | 대안 |
|-----------|------|------|
| 화면 구성 | 우측 슬라이드 패널 | 별도 페이지, 풀스크린 모달, 하단 슬라이드업 |
| 금액 단위 | 억원 단위 (12.3억) | 만원 단위, 원본+단위 표시 |
| 추세 차트 | 바 차트 | 라인 차트, 스파크라인 카드 |
| 메뉴 범위 | 전체 8가지 메뉴 탭 유지 | 재무계정+지표만, 요약 대시보드+상세 탭 |

## Technical Context

- **프론트엔드 스택**: Alpine.js + Tailwind CSS + Chart.js
- **기존 패턴**: 카드 내부 펼침식 패널 (`x-show`, `x-transition`)
- **API**: 기존 8가지 재무 API 엔드포인트 그대로 활용
- **금액 포맷**: `Format.number()` 유틸리티 함수 확장 필요
