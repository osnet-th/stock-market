아래 절차를 순서대로 수행해줘.

## 0. 설계 문서 경로 확인
- `$ARGUMENTS`가 비어있거나 없으면, `.claude/designs/` 하위의 설계 문서 목록을 검색해서 AskUserQuestion으로 선택지를 제시해.
- 사용자가 선택한 경로를 설계 문서 경로로 사용해.

## 1. 설계 문서 읽기
- 확인된 설계 문서 경로의 설계 문서를 읽어서 작업 리스트와 설계 내용을 파악해.

## 2. 변경 사항 분석
- `git status`와 `git diff --staged`, `git diff`를 실행해서 현재 변경된 파일 목록과 내용을 파악해.

## 3. 커밋 메시지 생성
- 설계 문서의 제목과 작업 리스트를 기반으로 커밋 메시지를 생성해.
- 커밋 메시지 형식:
  ```
  feat: {설계 문서 제목 요약}

  - {완료된 작업 1}
  - {완료된 작업 2}
  - ...

  설계 문서: {설계 문서 상대 경로}

  Co-Authored-By: Claude Opus 4.6 <noreply@anthropic.com>
  ```
- 커밋 타입은 변경 내용에 맞게 선택해 (feat, fix, refactor 등)

## 4. 스테이징 및 커밋
- 변경된 파일을 개별적으로 `git add` 해 (`.env`, credentials 등 민감 파일 제외)
- 생성한 커밋 메시지로 커밋해.

## 5. 푸시
- 현재 브랜치를 원격에 푸시해.

## 주의사항
- 커밋 전에 변경 파일 목록과 커밋 메시지를 보여주고 확인을 받아.
- 민감 파일(.env, credentials 등)은 절대 커밋하지 마.
