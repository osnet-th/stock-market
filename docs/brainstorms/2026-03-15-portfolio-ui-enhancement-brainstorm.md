# Portfolio UI Enhancement - Brainstorm

**Date:** 2026-03-15
**Status:** Decided

## What We're Building

포트폴리오 페이지의 상단 영역과 자산 섹션 헤더를 개선하여 전체 자산 비중을 시각적으로 파악하고, 각 자산 유형별 투자 성과를 한눈에 확인할 수 있도록 한다.

### 변경 범위

1. **상단 영역 재구성**: 기존 3개 요약 카드 + CSS 막대 차트 -> 도넛 차트 + 요약 정보 나란히 배치
2. **자산 섹션 헤더 강화**: 각 자산 유형별(현금 제외) 투자금액, 평가금액, 수익률 소계 표시

### 변경하지 않는 것

- 자산 항목의 개별 표시 방식 (기존 유지)
- 뉴스, 재무상세, 추가매수 등 기존 기능
- 백엔드 API (프론트엔드만 변경)

## Why This Approach

- **도넛 차트**: 포트폴리오 비중 시각화에 가장 직관적이며, 중앙에 총 평가금액을 표시하여 공간 효율적
- **Chart.js**: CDN으로 바로 사용 가능하고, 현재 프로젝트(Vanilla JS + Alpine.js)와 호환성 우수
- **섹션 헤더 소계**: 기존 UI 구조(접이식 섹션)를 유지하면서 자연스럽게 정보를 추가

## Key Decisions

| 결정 사항 | 선택 | 이유 |
|----------|------|------|
| 비중 차트 형태 | 도넛 차트 | 포트폴리오 비중 표현에 가장 직관적 |
| 차트 라이브러리 | Chart.js (CDN) | 경량, 간편, 도넛 차트 지원 우수 |
| 상단 카드 처리 | 도넛 차트 옆에 나란히 배치 | 비중 + 요약 정보를 한 영역에서 확인 |
| 항목별 소계 위치 | 자산 섹션 헤더 영역 | 기존 UI 구조 유지하며 자연스럽게 추가 |
| 현금 - 차트 | 포함 | 전체 자산 비중의 일부이므로 차트에는 표시 |
| 현금 - 소계 | 제외 | 투자금액/평가금액/수익률 소계 표시 불필요 |

## Technical Notes

- **영향 범위**: 프론트엔드만 (index.html, app.js)
- **Chart.js**: CDN (`<script src="https://cdn.jsdelivr.net/npm/chart.js">`)으로 추가
- **계산 로직**: 기존 `getEvalAllocation()`, `getTotalInvested()`, `getTotalEvalAmount()` 함수 활용
- **자산별 소계**: 기존 items 배열을 assetType으로 그룹핑하여 계산
- **수익률 계산**: 주식은 현재가 기반, 기타 자산은 investedAmount = evalAmount이므로 수익률 0%
