# Brainstorm: 주식 매수 시 원화(CASH) 자동 차감 연동 기능

**Date:** 2026-03-17
**Status:** Draft

## What We're Building

포트폴리오에서 주식을 추가/매수할 때, 등록된 원화(CASH) 항목을 선택적으로 연결하여 매수 금액만큼 원화 잔액이 자동 차감되는 기능.

**핵심 시나리오:**
1. 사용자가 "삼성증권 계좌" 원화(CASH) 항목을 등록 (1,000만원)
2. 주식 추가 시 해당 원화 항목을 선택하여 연결 (선택 사항)
3. 삼성전자 100주 x 70,000원 = 700만원 매수 → 원화 잔액 자동 300만원으로 차감
4. 같은 원화에 SK하이닉스도 연결 가능 (1:N 구조)

## Why This Approach

**별도 매핑 테이블(cash_stock_link) 방식 채택 이유:**
- PortfolioItem 도메인 모델 변경 없이 연결 관계 관리 가능
- 연결 이력 및 차감 이력 추적 가능
- 향후 확장성 (다른 자산 타입 연동 등) 확보
- 관심사 분리 — 연결/차감 로직이 독립적인 도메인으로 관리

## Key Decisions

| 결정 사항 | 선택 | 이유 |
|---|---|---|
| 적용 범위 | 주식(Stock)만 | 현재 주식 매수만 빈번, YAGNI 원칙 |
| 외화 주식 차감 기준 | investedAmountKrw | 사용자가 입력하는 원화 환산 금액 기준 |
| 잔액 부족 시 | 매수 차단 | 잔액 부족 시 에러 반환, 매수 불가 |
| 삭제 시 원화 복원 | 선택적 복원 | 복원 여부 선택 + 평가금액 기본값 제시 + 수동 입력 가능 |
| 연결 구조 | 1:N (원화 1 : 주식 N) | 하나의 계좌(원화)에서 여러 주식 매수하는 실제 사용 패턴 |
| 설계 방식 | 별도 매핑 테이블 | 도메인 분리, 확장성, 이력 관리 |

## Detailed Behavior

### 주식 추가 시
- 주식 등록 화면에서 "원화 연결" 드롭다운 표시 (등록된 CASH 항목 목록)
- 선택하지 않으면 기존과 동일하게 동작 (차감 없음)
- 선택하면 매수 금액(KRW 주식: quantity x purchasePrice, 외화 주식: investedAmountKrw) 차감
- 원화 잔액 부족 시 매수 차단 + 에러 메시지

### 추가 매수 시 (addStockPurchase)
- 이미 원화가 연결된 주식이면 추가 매수 금액도 자동 차감
- 원화 잔액 부족 시 매수 차단

### 주식 삭제 시
- 원화가 연결된 주식인 경우 "원화 복원" 옵션 제시
- 복원 선택 시: 현재 평가 금액 기준으로 "이 금액으로 복원합니다" 메시지 표시
- 사용자가 금액 수정 가능 (수동 입력)
- 복원 미선택 시: 원화 변동 없이 주식만 삭제

### 주식 수정 시 (updateStockItem)
- 연결된 원화가 있을 경우, 수정 전후 investedAmount 차이만큼 원화 자동 조정
- 예: 매수가 70,000 → 75,000 변경 시, 차액(수량 x 5,000원)만큼 원화 추가 차감
- 투자금액 감소 시: 차액만큼 원화 복원
- 잔액 부족 시: 수정 차단 (매수와 동일 규칙)
- 원화 연결 변경/해제도 수정 시 가능

### 원화 연결 해제/변경
- 주식 수정 화면에서 연결된 원화를 다른 원화로 변경하거나 "연결 없음"으로 해제 가능
- 해제 시: 기존 차감된 원화는 복원하지 않음 (이미 차감된 금액은 확정)
- 변경 시: 기존 원화 복원 없이, 이후 추가 매수부터 새 원화에서 차감
