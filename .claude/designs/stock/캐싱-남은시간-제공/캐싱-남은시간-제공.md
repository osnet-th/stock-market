# 주식 캐싱 남은시간 제공

> GitHub Issue: #19
> 모듈: stock

## 배경

현재 주식 현재가 조회 시 Caffeine 캐시로 30분간 데이터를 캐싱하지만, 사용자는 해당 데이터가 언제 캐싱된 것인지, 다음 갱신은 언제인지 알 수 없다. 사용자에게 캐싱 시점과 남은 시간 정보를 제공하여 데이터의 신선도를 인지할 수 있도록 한다.

## 현재 구조

- `KisStockPriceAdapter.getPrice()`: `@Cacheable`로 `StockPrice` 캐싱 (TTL 30분)
- `KisStockPriceAdapter.getDomesticPrices()`: 수동 `cache.get()`/`cache.put()`으로 캐싱
- `StockPriceResponse`: 캐시 관련 메타데이터 없음
- 프론트엔드: 캐시 정보 표시 없음

## 핵심 결정

### 1. 캐시 타임스탬프 추적 방식

`@Cacheable`은 캐시 저장 시점을 추적할 수 없으므로, `getPrice()`를 수동 캐시 관리로 전환한다. `getDomesticPrices()`에서 이미 수동 캐시를 사용하고 있어 일관성도 확보된다.

`ConcurrentHashMap<String, Instant>`으로 캐시 키별 저장 시점을 관리한다.

### 2. 캐시 정보 전달 방식

`StockPricePort`에 캐시 메타데이터 조회 메서드를 추가하지 않는다. 대신 Adapter 내부에서 캐시 정보를 포함한 래퍼를 반환하는 별도 메서드를 Service에서 호출한다.

- 새 record: `CachedStockPrice(StockPrice price, Instant cachedAt)`
- `StockPricePort`에 `CachedStockPrice getPriceWithCacheInfo(...)` 추가
- `getDomesticPrices()`도 `Map<String, CachedStockPrice>` 반환으로 변경

### 3. 응답 DTO 확장

`StockPriceResponse`에 3개 필드 추가:
- `cachedAt`: 캐싱 시점 (ISO 8601)
- `nextRefreshAt`: 다음 갱신 시점 (ISO 8601)
- `remainingSeconds`: 갱신까지 남은 초

### 4. 프론트엔드 표시

포트폴리오 현재가 영역에 캐시 정보 표시:
- "N분 N초 전 데이터 | M분 M초 후 갱신" 형태
- 1분 미만일 경우 초 단위로 표시 (예: "30초 전 데이터 | 29분 30초 후 갱신")

## 변경 대상

| 레이어 | 파일 | 변경 내용 |
|--------|------|----------|
| Domain | `CachedStockPrice.java` (신규) | `StockPrice` + `Instant cachedAt` 래퍼 record |
| Port | `StockPricePort.java` | `getPriceWithCacheInfo()`, `getDomesticPricesWithCacheInfo()` 추가 |
| Adapter | `KisStockPriceAdapter.java` | `@Cacheable` 제거, 수동 캐시 + 타임스탬프 관리 |
| DTO | `StockPriceResponse.java` | `cachedAt`, `nextRefreshAt`, `remainingSeconds` 필드 추가 |
| Service | `StockPriceService.java` | `CachedStockPrice` 사용, 캐시 정보 포함 응답 생성 |
| Frontend | `portfolio.js` | 캐시 남은시간 표시 로직 추가 |
| Frontend | `index.html` | 캐시 정보 UI 요소 추가 |

## 주의사항

- 기존 `getPrice()`, `getDomesticPrices()` 메서드의 시그니처는 유지하고, 새 메서드를 추가하는 방식으로 기존 호출에 영향 없도록 한다
- 캐시 TTL(30분)은 `StockPriceCacheConfig.STOCK_PRICE_CACHE_TTL` 상수로 추출하여 Adapter와 Response 생성 시 동일 값을 참조한다
- `cacheTimestamps`는 캐시 eviction 시 함께 정리되어야 하므로 `Caffeine.removalListener`를 활용한다
