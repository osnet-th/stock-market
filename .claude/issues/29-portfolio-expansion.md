기존 포트폴리오 서비스에 추가 기능을 개발하려고 한다.

현재 이미 있는 기능:
- 자산 등록
- 사용자가 어디에 투자되어 있는지 비율 표시
- 투자 수익률 계산
- 자산 삭제

따라서 기존 기능은 제외하고,
아래 추가 기능만 기능 요구사항 중심으로 정리해줘.
API 명세, DB 테이블 설계, 코드, SQL 등 구현 세부사항은 제외해줘.

추가하려는 기능은 다음과 같다.

1. 큰 자산 분류별 목표 비중 설정
- 특정 주식별 비중 계산은 필요 없다.
- 삼성전자 10%, NAVER 5% 같은 개별 종목 기준이 아니라,
  주식 40%, 현금 40%, 부동산 20% 같은 큰 자산 분류 기준으로 목표 비중을 설정한다.
- 자산 분류 예시는 주식, 현금, 부동산, 채권, ETF, 코인, 기타 등이 있다.
- 현재 자산 분류별 비중과 목표 비중을 비교해서 초과/부족/적정 상태를 보여줘야 한다.
- 목표 비중을 초과해도 자산 추가나 매수 등록을 막지는 않는다.
- 단, 화면에서는 목표 비중보다 초과했다는 표현을 보여줘야 한다.
- 예: 현재 주식 비중 45%, 목표 주식 비중 40%, 5%p 초과

2. 자산 분류별 초과/부족 표시
- 현재 비중, 목표 비중, 차이, 상태를 보여줘야 한다.
- 상태는 초과, 부족, 적정, 목표 미설정 등으로 표현한다.
- 가능하면 목표 비중에 맞추기 위해 줄이거나 늘려야 하는 참고 금액도 보여준다.
- 단, 투자 추천처럼 보이지 않도록 참고 정보로 표현한다.

3. 판매 기능 추가
- 현재는 삭제 기능만 있으나, 삭제와 판매는 분리되어야 한다.
- 삭제는 잘못 입력한 자산을 제거하는 기능이다.
- 판매는 실제 보유 자산을 판 기록을 남기는 기능이다.
- 판매는 일부 판매와 전량 판매를 지원해야 한다.

4. 판매 등록 시 기본값 자동 입력 및 수정 기능
- 현재 투자 수익률이나 평가금액은 이미 조회되고 있다고 가정한다.
- 다만 현재 조회값과 실제 판매 시점의 금액에는 차이가 있을 수 있다.
- 따라서 판매 등록 화면에서는 현재 조회된 값을 기준으로 판매금액 또는 판매단가를 자동으로 기본 입력해준다.
- 단, 사용자가 실제 판매금액에 맞게 판매금액 또는 판매단가를 직접 수정할 수 있어야 한다.
- 현재 조회된 값은 기본값일 뿐이며, 최종 저장 기준은 사용자가 확정한 판매금액이다.
- 수익금과 수익률은 사용자가 최종 입력한 판매금액을 기준으로 자동 계산되어야 한다.
- 사용자가 판매금액을 수정하면 수익금과 수익률도 즉시 다시 계산되어야 한다.

예시:
- 현재 조회 기준 예상 판매금액: 1,600,000원
- 사용자가 실제 판매금액을 1,580,000원으로 수정
- 수익금과 수익률은 1,580,000원을 기준으로 다시 계산
- 최종 저장되는 판매 수익금과 수익률도 1,580,000원 기준

화면 문구 예시:
- 현재 조회된 가격 기준으로 자동 입력되었습니다.
- 실제 판매금액과 다를 경우 수정할 수 있습니다.
- 수익금과 수익률은 입력한 판매금액 기준으로 자동 계산됩니다.

5. 판매 수익 기록
- 자산 판매 시 판매일, 판매 수량 또는 판매 금액, 판매 단가, 최종 판매금액, 수익금, 수익률, 판매 사유, 메모를 저장해야 한다.
- 판매 후에도 과거 판매 기록을 조회할 수 있어야 한다.
- 전량 판매한 자산도 과거 판매 내역에서는 확인 가능해야 한다.
- 판매 기록에는 해당 자산이 어떤 자산 분류였는지도 함께 보여주면 좋다.
- 저장되는 수익금과 수익률은 현재 조회값이 아니라 사용자가 최종 확정한 판매금액 기준이어야 한다.

6. 판매 타임라인
- 사용자는 언제 어떤 자산을 팔았고, 얼마를 벌거나 잃었고, 수익률이 얼마였는지 타임라인으로 볼 수 있어야 한다.
- 월별 또는 날짜별로 판매 내역을 보여주면 좋다.
- 자산 분류별로 필터링할 수 있으면 좋다.
- 예: 2026년 4월 / 삼성전자 일부 판매 / 주식 / +180,000원 / +12.9%

7. 판매 사유 기록
- 판매 시 사용자가 판매 사유를 선택하거나 직접 입력할 수 있으면 좋다.
- 예: 자산 분류 목표 비중 초과, 목표가 도달, 손절, 현금 확보, 리밸런싱, 기타

위 내용을 바탕으로 다음을 작성해줘.

1. 추가해야 하는 기능 목록
2. 기능별 사용자 시나리오
3. 화면에서 보여줘야 하는 정보
4. 사용자 경험상 주의할 점
5. MVP 우선순위
6. 있으면 좋은 추가 기능
7. 삭제와 판매를 혼동하지 않게 하는 UX 방향
8. 자산 분류별 목표 비중 초과 표현 예시
9. 판매 타임라인 표현 예시
10. 판매금액 수정 시 수익금/수익률 자동 재계산 UX

기존 기능은 이미 있다고 가정하고,
이번에 추가 개발해야 하는 기능만 정리해줘.
특정 주식별 목표 비중 계산은 제외하고,
큰 자산 분류별 목표 비중 관리 중심으로 작성해줘.
