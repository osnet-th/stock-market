# 국내 경제지표 대시보드 개선 브레인스토밍

## 무엇을 만들 것인가

현재 플랫 테이블로만 표시되는 국내 경제지표(ECOS) 화면을 **카드 대시보드 + 상세 테이블 하이브리드** 방식으로 개선한다.

### 현재 문제

1. **숫자만 나열**: 값이 좋은 건지 나쁜 건지, 전월 대비 어떤 변화인지 맥락이 없음
2. **항목 의미 불명확**: 지표명만 봐서는 경제에 어떤 의미인지 파악 어려움
3. **시각적 밀도**: 모든 행이 동일한 모양으로 나열되어 중요한 것과 덜 중요한 것 구분 불가

### 개선 후 모습

- **상단**: 카테고리별 핵심 지표 2~4개를 카드로 표시
  - 지표명 + 현재 값 + 전월 대비 변화(▲/▼ + 퍼센트/수치)
  - 카드 내 간단 해설 텍스트 (예: "기준금리가 인하되면 시중 유동성 증가")
  - 변화 방향에 따른 색상 코딩 (상승: 빨강/파랑, 하락: 파랑/빨강 - 지표 특성에 따라)
- **하단**: 기존 테이블 유지 + 가독성 개선
  - 분류별 그룹 헤더
  - 전월 대비 변화 컬럼 추가
  - 행 호버 시 툴팁으로 상세 설명

## 왜 이 접근 방식인가

- **하이브리드의 장점**: 한눈에 경기 상황 파악(카드) + 상세 데이터 접근(테이블) 모두 충족
- **전면 대시보드 대비**: 테이블의 정보 밀도를 유지하면서도 핵심 인사이트 제공
- **테이블만 개선 대비**: 근본적인 "한눈에 파악" 문제를 카드 영역으로 해결
- **기존 API 구조 활용**: 백엔드 변경을 최소화하면서 프론트엔드 중심 개선

## 핵심 결정사항

### 1. 전월 대비 변화 데이터

- ECOS API에서 직접 제공하지 않으므로 **이전 데이터를 DB에 저장하여 직접 계산**
- 배치 스케줄러에서 데이터 수집 시 이전 값도 함께 저장
- 변화율 = (현재값 - 이전값) / 이전값 * 100

### 2. 핵심 지표 선정 방식

- **1차**: 카테고리별 대표 지표를 코드로 사전 정의 (enum 또는 설정)
  - 예: 금리 → 기준금리, CD(91일), 통화/금융 → M2, 물가 → 소비자물가지수 등
- **향후**: 사용자 커스터마이징 (핀 고정) 기능 추가 가능

### 3. 색상 코딩 방향

- 전월 대비 변화 방향으로 표시 (▲ 상승 / ▼ 하락)
- 지표 특성에 따라 색상 의미 반전 필요 (물가 상승은 부정적, 성장률 상승은 긍정적)
- 지표별 `positiveDirection` 속성 필요 (UP 또는 DOWN)

### 4. 의미 설명 방식

- 카드 내: 지표명 아래 간단 해설 텍스트 상시 표시 (1줄)
- 테이블: 행 호버 시 툴팁으로 상세 설명

## Resolved Questions

1. **핵심 지표 목록**: 설계 단계에서 카테고리별 구체적 지표 목록을 확정한다.
2. **지표 해설 데이터**: 외부 설정 파일(YAML/JSON)로 관리하여 코드 변경 없이 수정 가능하도록 한다.
3. **변화 주기 차이**: 주기별 자동 대응 — 일간 → 전일 대비, 월간 → 전월 대비, 분기 → 전분기 대비 자동 적용.
