## What We're Building

데스크탑 위주로 설계된 Stock Market Dashboard를 태블릿과 모바일 브라우저에서도 정상적으로 사용할 수 있도록 반응형 웹 디자인을 적용한다. 모든 기능(조회, 등록, 편집)이 모바일에서도 동작해야 한다.

## Why This Approach

**Tailwind 반응형 유틸리티 중심 접근** 선택 이유:

- 현재 Tailwind CDN 기반 구조를 그대로 유지 (빌드 도구 도입 불필요)
- HTML에 `sm:`, `md:`, `lg:` 접두사 클래스 추가로 대부분 해결 가능
- custom.css에는 테이블→카드 전환 등 Tailwind만으로 어려운 부분만 최소한 media query 추가
- YAGNI 원칙 준수 — 최소 변경으로 최대 효과

## Key Decisions

| 결정 항목 | 선택 | 이유 |
|-----------|------|------|
| **대상 디바이스** | 태블릿 + 모바일 모두 | 외출 시에도 풀 기능 사용 목적 |
| **사이드바 (모바일)** | 햄버거 메뉴 + 오버레이 드로어 | 일반적인 모바일 네비게이션 패턴 |
| **챗봇 패널 (모바일)** | 전체 화면 모달 | 좁은 화면에서 대화 UX 최적화 |
| **테이블 (모바일)** | 카드형 레이아웃 전환 | 가로 스크롤보다 가독성 우수 |
| **브레이크포인트** | Tailwind 기본값 | sm:640px, md:768px, lg:1024px |
| **구현 방식** | Tailwind 반응형 유틸리티 중심 | 빌드 도구 변경 없이 기존 구조 유지 |

## Scope — 반응형 대응 영역

### 고위험 (레이아웃 깨짐)

1. **사이드바**: 고정 `w-56` → 모바일에서 숨기고 햄버거 버튼으로 오버레이 드로어 토글
2. **헤더**: 햄버거 버튼 추가, 사용자 정보 축소
3. **메인 콘텐츠**: `ml-56` → 모바일에서 `ml-0` 전체 너비 사용
4. **재무 슬라이드 패널**: `w-[65%]` → 모바일에서 전체 너비
5. **챗봇 패널**: `w-96` → 모바일에서 전체 화면 모달
6. **모달**: 고정 너비(`w-[480px]`, `w-96`) → `w-full max-w-[480px] mx-4`

### 중위험 (사용성 저하)

7. **포트폴리오 항목**: 액션 버튼 그룹 → 모바일에서 축소/드롭다운
8. **경제지표 테이블**: 테이블 → 모바일 카드형 레이아웃 전환
9. **요약 카드 그리드**: `md:grid-cols-4` → 이미 반응형이나 터치 영역 확인 필요

### 저위험 (이미 대응 또는 변경 최소)

10. **로그인/회원가입 페이지**: 이미 중앙 정렬, 반응형에 가까움
11. **Chart.js 차트**: `responsive: true` 설정으로 자동 리사이징
12. **탭 네비게이션**: `overflow-x-auto`로 스크롤 가능

## 기술적 고려사항

- **터치 이벤트**: 챗봇 드래그 기능에 `touchstart`/`touchmove` 이벤트 추가 필요
- **Alpine.js 상태**: `sidebarCollapsed`를 viewport 기반으로 자동 전환하는 로직 추가 (window.matchMedia 또는 resize 리스너)
- **CSS 전략**: Tailwind 유틸리티 클래스 우선, custom.css에는 테이블→카드 전환 등 복잡한 패턴만 media query 추가
