# AI 채팅 페이지 컨텍스트 초기화

**Date:** 2026-04-08

## What We're Building

AI 채팅이 페이지 이동 시 이전 컨텍스트(모드, 대화 내역)를 유지하는 문제를 해결한다.

**현재 문제:** ECOS 페이지에서 ECONOMIC 모드로 AI 채팅을 열고 포트폴리오 페이지로 이동하면, AI 채팅 버튼을 눌렀을 때 여전히 ECONOMIC 모드의 이전 대화가 표시된다.

**목표 동작:**
1. 페이지 이동 시 열려있는 채팅을 자동으로 닫는다
2. 채팅 버튼 클릭 시 현재 페이지에 맞는 모드로 초기화한다 (이전 대화 리셋)
3. AI 채팅 버튼은 ECOS, 포트폴리오 페이지에서만 표시한다 (home, keywords, global에서는 숨김)

## Why This Approach

- **페이지 컨텍스트 불일치 방지:** 포트폴리오 페이지에서 경제지표 분석 대화가 보이는 것은 UX 혼란
- **간결한 구현:** 기존 `navigateTo()`와 `toggleChat()`에 최소한의 로직 추가로 해결 가능
- **명확한 진입점:** 각 페이지에서 채팅을 열 때 해당 페이지에 맞는 모드로만 시작하여 사용자 혼란 제거

## Key Decisions

| 결정 | 선택 | 이유 |
|------|------|------|
| 전환 시점 | 채팅 버튼 클릭 시 | 페이지 이동 중 불필요한 상태 변경 최소화 |
| 페이지 이동 시 채팅 | 자동 닫힘 | 다른 페이지 컨텍스트의 대화가 남아있는 것 방지 |
| 이전 대화 | 초기화 | 모드 전환 시 이전 대화는 컨텍스트가 맞지 않으므로 리셋 |
| 버튼 표시 범위 | ECOS, 포트폴리오만 | 다른 페이지에는 아직 AI 분석 모드가 없음 |
| global 페이지 | 버튼 숨김 | 향후 필요 시 추가 |

## 영향 범위

### 수정 대상 파일
- `static/js/app.js` - `navigateTo()`: 페이지 이동 시 채팅 닫기
- `static/js/components/chat.js` - `toggleChat()`: 현재 페이지 기반 모드 결정 로직
- `static/index.html` - AI 채팅 버튼: 페이지별 표시/숨김 조건 추가

### 페이지별 채팅 모드 매핑

| global | - | 숨김 |
