# 추가 납입 기능 브레인스토밍

**날짜**: 2026-04-10
**상태**: 브레인스토밍 완료

---

## What We're Building

포트폴리오의 현금성 자산(예금/적금/CMA)과 펀드에 대한 **추가 납입 이력 관리 기능**.

### 핵심 기능
- **납입 이력 기록**: 날짜, 금액, 메모를 수동 입력하여 납입 내역 추적
- **자동납입 설정**: 월 납입액, 납입일을 설정하여 자동납입 예정 관리
- **총 투자금액 자동 갱신**: 납입 이력 합계로 `investedAmount` 자동 계산
- **만기 예상 금액 계산**: 적금의 경우 이자율 기반 만기 시 예상 수령액 계산

### 대상 자산 유형
- 예금 (DEPOSIT)
- 적금 (SAVINGS)
- CMA
- 펀드 (FUND)

---

## Why This Approach

**방식 1: 납입 이력 중심** 선택 이유:

1. **기존 패턴 활용**: `StockPurchaseHistory`와 동일한 패턴으로 `DepositHistory` 구현 → 코드 일관성 유지
2. **적절한 복잡도**: Quartz 자동 스케줄(방식 2)은 실제 납입 여부를 확인할 수 없어 과도하고, MVP(방식 3)는 이력 추적이 불가하여 부족
3. **확장 가능**: 이력 기반이므로 향후 통계/리포트 기능 확장 용이

---

## Key Decisions

1. **`DepositHistory` 엔티티 신규 생성** — `PortfolioItem`과 ID 기반 참조 (JPA 연관관계 없음, 프로젝트 규칙 준수)
2. **자동납입 설정은 필드 추가로 해결** — `CashDetail`/`FundDetail`에 `monthlyDepositAmount`, `depositDay` 필드 추가 (별도 엔티티 불필요)
3. **만기 예상 금액은 도메인 로직으로 구현** — Entity가 아닌 도메인 모델 또는 서비스에서 이자 계산
4. **UI는 기존 Vanilla JS 패턴** — 포트폴리오 상세 화면에 납입 이력 탭/섹션 추가
5. **납입 시 `investedAmount` 자동 갱신** — 납입 이력 추가/삭제 시 `PortfolioItem`의 투자금액 동기화

## Resolved Questions

1. **자동납입 미납 처리** → **미납 표시**: UI에서 납입 예정일이 지났음을 시각적으로 표시 (미납 배지 등)
2. **펀드 납입과 좌수 관계** → **금액 + 좌수 함께 기록**: 펀드 납입 시 납입 금액과 취득 좌수를 모두 기록
